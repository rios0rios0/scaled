# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run

```bash
yarn install              # install dependencies (Yarn 4.x Berry)
yarn build                # compile TypeScript → lib/
yarn prepack              # full build: clean lib/, compile, generate oclif manifest + README
yarn lint                 # ESLint with airbnb-typescript/base
yarn lint:fix             # auto-fix lint issues
yarn test                 # mocha + nyc coverage
./bin/run start <service> # run CLI (requires prior build)
```

`yarn prepack` must run before `./bin/run` — the CLI loads compiled JS from `lib/`.

## Architecture

TypeScript CLI built on oclif v1 (`@oclif/command`). Orchestrates security scanners (Nmap, Nikto, SQLMap) via Docker Compose.

- **DDD layout**: `src/domain/` holds entities, repository contracts, and service interfaces. `src/infrastructure/` has concrete implementations. Business logic stays decoupled from I/O.
- **Strategy pattern**: `src/resolver/strategy/` provides pluggable execution strategies (local vs. remote).
- **Service builder**: `src/resolver/service-builder.ts` constructs `ServiceDefinition` objects (name + path to a tool directory under `tools/`).
- **Manager**: `src/manager/` handles the start → report → stop lifecycle via Listr task lists.
- **Docker-first**: each scanner lives under `tools/<name>/` with its own `docker-compose.yml`.

Single command: `scaled start <service> [--build] [--containers <n>] [--set-env KEY=VALUE]`.

## Conventions

- ESLint extends `airbnb-typescript/base` with overrides: `class-methods-use-this`, `no-await-in-loop`, `no-restricted-syntax`, `no-continue` all disabled.
- Strict TypeScript (`tsconfig.json`). Target ES2017, output to `lib/`.
- README is auto-generated by `oclif-dev readme` during `prepack` — do not edit manually.
- Commits follow [Conventional Commits](https://www.conventionalcommits.org/) per the project's [Git Flow guide](https://github.com/rios0rios0/guide/wiki/Life-Cycle/Git-Flow).
